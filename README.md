# 인공위성 천체 관측 방해 예측기 — GitHub Pages 버전

지구과학Ⅱ의 적도좌표계를 실제 인공위성 관측 방해 문제에 적용하는 정적 웹앱입니다.

## 왜 GitHub Pages인가

기존 Streamlit 버전은 GitHub 계정을 제3자 서비스와 연결해야 합니다. 이 버전은 HTML, CSS, JavaScript만 사용하므로 GitHub Pages에서 직접 실행할 수 있습니다.

GitHub Pages는 Python을 실행하지 않으므로 기존 `app.py`가 아니라 아래 파일을 사용합니다.

```text
index.html
style.css
app.js
.nojekyll
README.md
```

## 배포 방법

1. 저장소에 있는 기존 파일을 삭제하거나 다른 폴더로 옮깁니다.
2. 이 폴더 안의 파일을 저장소 최상위에 업로드합니다.
3. GitHub 저장소의 `Settings`를 엽니다.
4. 왼쪽 메뉴에서 `Pages`를 누릅니다.
5. `Build and deployment`의 Source에서 `Deploy from a branch`를 선택합니다.
6. Branch는 `main`, 폴더는 `/(root)`를 선택합니다.
7. `Save`를 누릅니다.
8. 수 분 뒤 `Visit site`를 누릅니다.

## 앱 사용 방법

1. `최신 Starlink 데이터 불러오기`를 누릅니다.
2. 자동 불러오기가 실패하면 CelesTrak OMM JSON 파일을 내려받아 `OMM JSON 파일 선택`으로 불러옵니다.
3. 관측 조건을 설정합니다.
4. `위성 통과 위험 분석`을 누릅니다.
5. 추천 촬영 일정과 위험 사건을 CSV로 저장합니다.

## 분석 원리

- OMM 궤도 요소를 satellite.js의 SGP4 모델로 전파합니다.
- 관측 천체의 적경·적위를 관측지 기준 고도·방위각으로 변환합니다.
- 위성과 천체의 구면 각거리를 계산합니다.
- 위성이 관측 시야 안에 들어오며 지평선 위에 있고 태양빛을 받는 시간을 위험 후보로 분류합니다.
- 기본 촬영 시각을 앞뒤로 이동하여 겹치는 위성 수가 가장 적은 시각을 추천합니다.

## 한계

- 위성의 실제 겉보기등급을 계산하지 않습니다.
- 지구의 원통형 본그림자를 이용한 근사 판정입니다.
- 기상, 광공해, 건물, 산, 센서의 포화 특성을 반영하지 않습니다.
- 전체 위성군이 아니라 선택한 표본에 대한 탐색적 분석입니다.
